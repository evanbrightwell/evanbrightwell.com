# Site Strategy

## Positioning

This site is not only a portfolio. It is a personal design system for Evan's
work, and it should eventually connect to a refreshed resume.

The core message:

> Evan builds 3D visual work for spaces, displays, products, and process-led
> systems. He can make the image, but he can also shape the approach, organize
> the problem, and develop workflows beyond straightforward execution.

## Primary Experience

The site should move from fast impression to deeper proof:

1. Home: immediate identity, visual quality, and category orientation.
2. Category pages/sections: a clear point of view for each type of work.
3. Project entries: strongest examples with enough context to understand role,
   purpose, software, and outcome.
4. Process: a differentiated section for systems, workflows, tools, and methods.
5. Bio/contact: credibility and next step.

## Category Roles

### Spaces

Spaces should present environmental and experiential work as concept tools.

Use for:

- bars
- lounges
- suites
- dinners
- activations
- hospitality and event environments

The page should communicate spatial thinking, atmosphere, lighting, material,
and the ability to make future spaces feel believable before they exist.

### Displays

Displays should cover the object/display layer between product and space.

Use for:

- bottle glorifiers
- retail fixtures
- carts
- trays
- tabletop display moments
- sculptural product presentation pieces

The page should communicate product presence, physical brand moments, material
detail, and display systems.

### Products

Products should be controlled, object-forward, and image-led.

Use for:

- kits
- packaging
- bottle renders
- launch imagery
- serveware
- product compositions

The page should communicate precision, lighting, material fidelity, staging,
and clean client-ready imagery.

### Process

Process should be the differentiator.

Use for:

- procedural tools
- image/workflow systems
- motion experiments
- sculpting
- custom methods
- selected archive work only when it supports the larger story

This page should not become a miscellaneous archive. It should explain how Evan
thinks and builds systems that solve visual problems.

## Navigation Rules

- Category controls should change work category.
- Site navigation should change site mode or section.
- Do not show multiple controls that appear to do the same job unless their roles
  are visually and behaviorally distinct.
- Any visible waypoint or button must have a real destination or clearly disabled
  state.
- The home page should remain a focused first-screen experience, not a full
  scroll dump of every section.

## Project Card Requirements

Minimum fields:

- title
- category
- image
- purpose
- software
- role

Preferred fields:

- client or brand context
- year
- lane/subcategory
- strongest image set
- short project story
- whether it should become a full project page

## Resume Connection

The resume should eventually use the same language:

- Spaces: environmental, experiential, hospitality, activation visualization.
- Displays: fixtures, glorifiers, retail tools, physical brand moments.
- Products: packaging, product imagery, kits, launch visuals.
- Process: workflows, procedural systems, tools, technical/creative problem solving.

This creates one coherent professional story instead of separate website and
resume identities.
