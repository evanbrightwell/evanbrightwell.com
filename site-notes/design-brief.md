# Design Brief

## Goal

Create a personal website for Evan Brightwell that acts as a polished personal
presence, portfolio, and foundation for a future resume refresh.

The site should make visitors come away thinking:

> Evan is a 3D artist/designer we should talk to. He has strong visual taste,
> thinks in systems, and can solve problems beyond simply making the requested
> image.

## Primary Audience

- Clients considering Evan for freelance/project work.
- Employers or collaborators evaluating visual taste, thinking, and range.
- People who need a quick but credible sense of Evan's 3D/design practice.

## Homepage

The first screen should include:

- Evan Brightwell's name.
- A short blurb.
- Entry points into the primary work categories.
- A visual or interaction system that makes the site feel custom.

The home page should stay focused. It should not expose every section in a long
scroll by default.

## Sections

- Bio
- Gallery
- Work
- Contact

## Primary Work Categories

- Spaces: rooms, lounges, suites, bars, dinners, hospitality environments, and activations.
- Displays: glorifiers, fixtures, carts, trays, retail tools, and sculptural product displays.
- Products: kits, packaging, bottle renders, launch assets, serveware, and product-led compositions.
- Process: procedural systems, tools, workflows, motion, sculpting, and selected experiments.

These categories should also inform the future resume structure.

## UX Direction

- Top-level controls should have clear jobs.
- Category navigation should answer "what kind of work am I looking at?"
- Site navigation should answer "where am I in Evan's practice?"
- Avoid controls that look different but do the same thing.
- Avoid clickable points that do not go anywhere.
- Project cards should eventually contain title, category, client/context,
  purpose, software, role, and a stronger project story when available.

## Avoid

- Blog or personal writing.
- Generic centered hero card portfolio template.
- Overly flashy interaction that gets in the way.
- One-note palettes dominated by purple, beige, dark blue, brown, or orange.

## Visual Concepts Explored

1. Refined Interface Portfolio
2. Interactive Gallery Navigation
3. Minimal Personal System

The final direction may combine elements from all three.

## Current Strategic Questions

- What does each category page do beyond showing a carousel?
- Which projects deserve full project pages versus gallery-only treatment?
- What makes Process feel like a differentiator instead of a miscellaneous archive?
- How should the website and resume share language, categories, and hierarchy?
